# V0RTEX Plugin Policy

This is the policy reference for the [V0RTEX Plugin Hub](https://github.com/Vider06/V0rtex-Plugin-Official).

The full technical documentation — plugin classes, permissions, code rules, API access, header format, lifecycle hooks, examples, and enforcement details — lives in the main V0RTEX repository:

> 📄 **[PLUGINS.md — github.com/Vider06/V0rtex](https://github.com/Vider06/V0rtex/blob/main/PLUGINS.md)**

In case of conflict, PLUGINS.md is authoritative.

---

## Quick Reference

### Plugin Classes

| Class | Description |
|---|---|
| 🔴 V0RTEX-Made | Written by Vider_06. Full V0RTEX access. Not open to community. |
| ⚡ Elevated | Verified + explicit special permissions granted by D.V.V. |
| ✅ Verified | Reviewed, signed, limited read-only V0RTEX API access. |
| ⚠️ Unverified | Not reviewed. No V0RTEX access. Warning shown before loading. |
| 🚫 Banned | Permanently blocked in all V0RTEX versions. No override. |

### Enforcement Levels

| Level | Name | Effect | Fix Window |
|---|---|---|---|
| 🟡 1 | Warning | Plugin loadable, badge shown | 30 days or → Level 2 |
| 🟠 2 | Suspension | Plugin blocked | 14 days or → Level 3 |
| 🔴 3 | Permanent Plugin Ban | Plugin banned forever | Appeal within 7 days only |
| ⛔ 4 | Permanent Account Ban | Account + all plugins banned | No appeal. Ever. |

### Banned Plugins

Maintained in the private repository [V0rtex-Banned-Plugins](https://github.com/Vider06/V0rtex-Banned-Plugins), accessible only to the D.V.V. V0RTEX checks against it at every startup.
