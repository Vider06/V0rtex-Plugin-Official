# Security Policy — V0RTEX Plugin Hub

## Scope

This policy covers security vulnerabilities in:

- The V0RTEX Plugin Hub repository infrastructure
- The `manifest.json` plugin index
- The Plugin Manager (`plugin_manager.py`) when released

It does **not** cover vulnerabilities in individual community plugins — report those using the standard plugin report process (Discussions → 🚨 Report).

For vulnerabilities in V0RTEX itself, refer to the security policy in the [main V0RTEX repository](https://github.com/Vider06/V0rtex).

---

## Reporting a Vulnerability

**Do not open a public Discussion or Issue for security vulnerabilities.**

Contact [@Vider06](https://github.com/Vider06) directly via GitHub private message or email (if listed on profile).

Include:

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

---

## Response

The D.V.V. will acknowledge the report and assess severity. If confirmed, a fix will be issued as soon as possible. You will be credited in the relevant changelog entry unless you prefer to remain anonymous.
